A Video Review

“Don’t Fear the Internet”

by Russ Maschmeyer (aka ‘Strange Native’)

My Thoughts

The Internet influences many aspects of our daily lives in society today. As such, it is important to know how the Internet and websites work. Ignorance is no excuse when it come to a vastly used resource and commodity.

I really enjoyed the "Don't Fear the Internet" tutorial videos. Russ Maschmeyer (aka ‘Strange Native’) has a very articulate voice and does an excellent job of explaining and teaching about the Internet, websites, HTML, CSS, and more. I would definitely recommend these videos to anyone attempting to learn HTML or CSS, or to anyone who simply wants to learn more about how website work and are designed. The tutorials can empower non-web designers and make them more comfortable with the platform they are now manipulating.

These tutorial videos reaffirmed wheat I have been learning in Web Design class. Furthermore, Russ Maschmeyer has taught me that I don't need to be a designer, photographer, or creative person to build my own (good-looking) website.

Three Things I learned:

  1. A website is a series of files stored on a server, located at a web address, that anyone can find with a computer. There are two kinds of websites: static and dynamic.
  2. CSS has a limited set of properties that can be used to style (just like HTML has a limited set of elements that define content types).
  3. Pseudo elements can be added on to the selector properties used in CSS to style specific parts or properties of an element.

“10 Things I Didn’t Know About HTML”

by Mathias Bynens

My Thoughts

In his presentation, “10 Things I Didn’t Know About HTML”, Mathias Bynens explained how certain elements, attribute entities, notations, and tags can be used or omitted to make HTML documents valid.

Mathias Bynens asserts that it is important to have a HTML or CSS validation website check your written code, it is also crucial to have a human check it as there are many mistakes that can be made which will be read as valid by a program but are incorrect. For example, quoted attributes, elements not included in DTD, and end tags.

In my opinion, this video was very complex and used language that I, a non-web designer, didn't understand much of the language was complicated hard to follow. It was like reading/listening to a different language (which it kind of is). But, nonetheless, it was a very informative video and I did learn quite a lot about HTML that I didn't know before.

Three Things I learned:

  1. Browsers will automatically render tags (which are optional in the mark up). Therefore, you can have a valid HTML document with missing end tags.
  2. Quotes, single quotes, and unquoted attribute values are all valid in HTML if an attribute value has no spaces.
  3. DTD (Document Type Definition) contains the elements, attribute entities, notations, and tags used in a HTML document. Without using DTD a HTML document would not be valid.

“Front End Legos: Better Design with Reusable HTML & CSS”

by Shay Howe

My Thoughts

CSS code is a method of separating presentation and content. This separation improves content accessibility, provide more control of presentation characteristics, enable multiple HTML pages, and reduce complexity and repetition.

In Shay Howe's presentational video, "Front End Legos: Better Design with Reusable HTML & CSS", he asserts that code must be maintained through organized and modular performance. Performance allows the reuse of code, not the duplication but the removal of old code. Its interesting that Shay Howe says to stay away from copying and pasting code when you can organize and modulate a more concise way of presentation - someone could have separate CSS documents for each page of his/her website as to allow a lower specificity when someone requests and loads to page or combine similar CSS files.

I had conflicted feelings about this video. I understand what I have learned about HTML and CSS. However, Shay Howe presents some very interesting ideas about how to better use HTML and CSS to web design. I don't know if it would be worthwhile to change CSS for all those who know and use it, but standards need to evolve (like technology and language) to help future designers in their fields.

Three Things I learned:

  1. Specificity determines which CSS styles are applied to an element.
  2. Data URI is a Uniform Resource Identifier scheme that creates performance enhancement, regardless of Internet connection, by removing a browsers limited requests that become stuck in a bottleneck of demanded data.
  3. Specificity can be measured by IDs, Classes/Pseudo-classes/Attributes, and Elements.. High specificity is applied before low specificity, but the goal is to have the lowest specificity through unnecessary nesting.
